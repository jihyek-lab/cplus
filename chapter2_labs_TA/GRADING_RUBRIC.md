# GRADING_RUBRIC (미니과제 채점 포인트)

각 미니과제는 10점 만점(권장)으로 채점할 수 있습니다.
- 6점: 요구 기능 구현(핵심 요구 충족)
- 2점: 코드 가독성(변수명/주석/출력 메시지 등)
- 2점: 예외/실수 방지(필요한 경우만: 입력 버퍼 처리 등)

---

## 01 (자기소개 2줄 출력)
- (6) 추가 출력 2줄 이상
- (2) 문장 형태로 출력(가독성)
- (2) 컴파일/실행 정상

## 02 (double로 나눗셈 소수 출력)
- (6) double 변환 사용: `static_cast<double>(a)/b` 등
- (2) 정수 나눗셈과 비교 출력하면 가산점(선택)
- (2) 출력 메시지 명확

## 03 (std:: 붙여서 컴파일)
- (6) `using namespace std;` 제거 후 `std::`로 정상 컴파일
- (2) 필요한 include 유지
- (2) 출력 동일/유사

## 04 (alice 네임스페이스 추가)
- (6) 새 namespace 추가 + 3개 함수 모두 호출
- (2) 출력으로 구분 가능
- (2) 모호성(ambiguous) 없이 컴파일

## 05 (둘레 추가)
- (6) perimeter 계산: `2*(w+h)` 정확
- (2) area와 perimeter 모두 출력(가독성)
- (2) 입력 안내 메시지 적절

## 06 (정수 입력까지 반복)
- (6) 반복 구조(while/do-while)로 정수 입력까지 반복
- (2) 실패 시 clear+ignore 처리
- (2) 성공 시 값 출력 후 종료

## 07 (name+age 출력)
- (6) `char name[]` + `int age` 입력받아 출력
- (2) 출력 형식 깔끔
- (2) 배열 크기 고려(20) 언급하면 가산점

## 08 (문장 2번 출력)
- (6) getline으로 입력받고 동일 문장 2회 출력
- (2) 출력 라벨 있으면 가산점
- (2) 컴파일 정상

## 09 (age+name+hobby 확장)
- (6) age 입력 후 ignore 처리, name/hobby getline 정상
- (2) 출력이 모두 포함
- (2) 입력 안내 메시지 명확

## 10 (단어 2개 입력 받아 word1-word2)
- (6) string 2개 입력받아 결합 출력(대시 포함)
- (2) 예외 없이 입력 동작
- (2) 출력 가독성

## 11 (가수+노래 한 줄 입력)
- (6) getline 2번 사용(공백 포함)
- (2) 라벨 포함 출력
- (2) 버퍼 문제 방지(필요 시 ignore)

## 12 (비밀번호 3회 제한)
- (6) 최대 3회 시도, 성공 시 즉시 종료
- (2) 남은 횟수 안내
- (2) 3회 실패 시 종료 메시지

## 13 (관리자 비번 먼저 받고 비교)
- (6) adminPw 입력받고, userPw 입력받아 비교
- (2) 입력 안내/출력 가독성
- (2) 논리 흐름 명확

## 14 (숫자 2개, 각각 sqrt 출력)
- (6) double 2개 입력, sqrt 각각 출력
- (2) 출력 라벨
- (2) 음수 입력 시 NaN 가능성 언급하면 가산점(선택)
