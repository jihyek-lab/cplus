핵심만 맞으면 정답.

## 정적 바인딩 vs 동적 바인딩
- virtual이 없으면 포인터 타입(Base*) 기준으로 컴파일 타임에 결정(정적 바인딩).
- virtual이 있으면 실제 객체 타입 기준으로 런타임에 결정(동적 바인딩).

## override
- 오버라이딩 시그니처 불일치(오타, const 차이 등)를 컴파일 단계에서 잡아준다.

## Base::func 강제 호출
- scope resolution(Shape::draw)로 동적 바인딩을 우회하고 부모 구현을 강제 호출.

## 가상 소멸자
- Base*로 Derived를 delete할 가능성이 있으면 Base 소멸자는 virtual이어야 한다.
- 아니면 Derived 소멸자가 호출되지 않아 자원 누수가 발생할 수 있다.

## 순수 가상 함수/추상 클래스
- virtual f() = 0; 는 순수 가상 함수.
- 순수 가상 함수를 가진 클래스는 추상 클래스 -> 객체 생성 불가.
- 포인터/참조는 가능, 파생 클래스가 모두 구현하면 사용 가능.
