## 채점 원칙

- 각 문항은 시험지에 제시된 `답안 가이드`의 항목별로 체크한다.
- 표현이 달라도 핵심 개념이 맞으면 정답으로 인정한다.
- 결과만 맞고 이유가 없으면 부분점만 준다.
- 코드 작성은 의도와 결과가 같으면 정답으로 처리한다.

---

## 1번. 입력 버퍼와 getline (10점)

- 원인 설명: `cin >>` 뒤에 개행이 버퍼에 남는다고 설명 4점
- 남는 문자: `\n` 명시 2점
- 해결 코드: `cin.ignore(...)` 또는 같은 취지 코드 4점

---

## 2번. C-문자열과 string 비교 (10점)

- `char[]`는 문자 배열, `string`은 문자열 클래스라고 설명 4점
- `strcmp()`로 내용을 비교해야 한다고 설명 3점
- 공백 포함 입력은 `getline()` + `string`이 적절하다고 설명 3점

---

## 3번. 캡슐화와 접근 지정자 (10점)

- 캡슐화 의미 설명 3점
- 멤버 변수를 `private`로 두는 이유 설명 4점
- 외부 사용용 함수를 `public`으로 두는 이유 설명 3점

---

## 4번. 생성자와 소멸자 호출 순서 (10점)

정답 출력:

```text
C
C
D
C
D
D
```

- 출력 결과 정확 4점
- 생성자 호출 시점 설명 3점
- 소멸자 호출 시점 및 스코프 종료 설명 3점

---

## 5번. 객체 배열과 기본 생성자 (10점)

- 배열의 각 원소 객체가 생성된다고 설명 4점
- 인수 없는 생성이 필요할 수 있다고 설명 3점
- 기본 생성자가 없으면 컴파일 문제가 생길 수 있다고 설명 3점

---

## 6번. 동적 메모리와 this 포인터 (10점)

- `new[]`에는 `delete[]`가 필요하다고 설명 3점
- `delete`만 쓰면 잘못된 해제라고 설명 2점
- `this`는 현재 객체를 가리킨다고 설명 2점
- `this->radius = radius;`에서 왼쪽은 멤버, 오른쪽은 매개변수라고 설명 3점

---

## 7번. 값 전달 / 주소 전달 / 참조 전달 비교 (10점)

- 값 전달: 원본 변경 안 됨 3점
- 주소 전달: 원본 변경 가능 2점
- 참조 전달: 원본 변경 가능 2점
- 호출 문법 차이 설명 1점
- 적절한 사용 상황 제시 2점

---

## 8번. 복사 생성자와 깊은/얕은 복사 (10점)

- 복사 생성자 호출 상황 2가지: 각 2점, 총 4점
- 얕은 복사 위험 설명 3점
- 깊은 복사 설명 3점

---

## 9번. 오버로딩, 디폴트 매개변수, 모호성 (10점)

- 오버로딩 구분 기준: 매개변수 개수/타입 3점
- 리턴 타입만 다른 함수 불가 이유 2점
- 디폴트 매개변수 규칙: 뒤에서부터 연속 2점
- 모호성 발생 이유 설명 3점

---

## 10번. static 멤버와 짧은 코드 작성 (10점)

빈칸 정답:

```cpp
count++;
count--;
```

- 첫 번째 빈칸 2점
- 두 번째 빈칸 2점
- `count`는 클래스 전체가 공유하는 값 설명 3점
- static 함수에는 `this`가 없어서 non-static 멤버에 직접 접근할 수 없다고 설명 3점
