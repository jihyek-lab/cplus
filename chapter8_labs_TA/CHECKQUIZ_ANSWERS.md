# CHECKQUIZ_ANSWERS (8장)

핵심만 맞으면 정답 처리.

## 상속 기본
- 파생 객체는 기본 클래스 부분을 포함한다(메모리/상태가 포함).
- private는 파생 클래스 내부에서도 직접 접근 불가(캡슐화).
- 업캐스팅은 파생->기본으로 안전(기본 부분은 항상 존재)하여 자동.
- 다운캐스팅은 기본->파생으로 위험(실제 객체가 파생이 아닐 수 있음)하여 명시적 캐스팅 필요.

## protected
- 클래스 내부/파생 클래스 내부에서 접근 가능, 외부에서는 불가.
- '객체 타입'이 아니라 '코드 위치'가 접근 권한을 결정한다.

## 생성/소멸
- 생성: Base -> Derived
- 소멸: Derived -> Base

## 기본 생성자 문제
- 파생 생성자는 기본적으로 Base()를 호출하려 함.
- Base()가 없으면 파생 생성자 초기화 리스트에서 Base(int) 등 명시해야 함.

## 상속 접근 지정자
- public 상속: Base public 유지
- protected 상속: Base public이 protected로 내려감
- private 상속: Base public/protected가 private로 내려감

## 다중/가상 상속
- 다이아몬드에서 A가 2번 포함되면 멤버 접근이 모호해질 수 있음.
- virtual 상속으로 A를 1개만 공유하도록 강제.
