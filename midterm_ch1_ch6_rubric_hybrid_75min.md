## C. 코드 분석 / 오류 수정

### 15번. [중] (5점)

정답 요지:

- `cin >> age` 뒤에 개행 문자 `\n`가 버퍼에 남음
- `getline()`이 그 개행을 바로 읽어 빈 줄이 됨
- 수정 코드 예: `cin.ignore();`

배점:

- 원인 설명 3점
- 수정 코드 2점

허용:

- `cin.ignore(1, '\n');`

### 16번. [중] (6점)

정답 요지:

- `new int[3]`은 배열 할당이므로 `delete p;`가 아니라 `delete[] p;`가 맞음
- 메모리를 해제한 뒤 `p[1]`에 접근하면 해제된 메모리를 읽게 되어 위험함

배점:

- 문제 1 설명 2점
- 문제 2 설명 2점
- 수정안 2점

허용되는 수정 예:

- `delete[] p;`로 고치고, `cout << p[1] << '\n';`를 `delete[] p;` 앞로 옮김
- 출력 후 `delete[] p;` 수행

### 17번. [중상] (6점)

정답 요지:

- `sum(3)`: 오류
- `sum(3, 4)`: 정상
- `sum('a')`: 정상

배점:

- `sum(3)` 판단 3점
- `sum(3, 4)` 판단 1점
- `sum('a')` 판단 1점

### 18번. [중상] (4점)

정답 요지:

- static 함수에는 `this`가 없음
- `money`는 객체마다 다른 non-static 멤버라 직접 접근 불가

배점:

- 오류 원인 설명 3점
- 수정 방법 1가지 제시 2점

허용되는 수정 예:

- `static` 제거
- `Person&` 또는 `const Person&`를 인자로 받게 수정
- `money`를 static으로 바꾸기

소계: 21점

## D. 출력 예측 / 심화 분석

### 19번. [중상] 복사 생성자 호출 분석 (4점)

정답:

- `출력 결과`: `ABB`

배점:

- 출력 결과 5점

### 20번. [중상] 참조 반환의 위험 (6점)

정답 요지:

- 지역 변수 `x`는 함수 종료와 함께 사라짐
- 사라진 지역 변수에 대한 참조를 반환하므로 위험함

배점:

- 문제점 설명 3점
- 수정 방법 1가지 제시 3점

허용되는 수정 예:

- 반환 타입을 `int`로 바꾸기
- 지역 변수가 아니라 외부에서 전달된 변수나 정적 변수를 참조하게 수정

### 21번. [상] 깊은 복사 코드 수정 (8점)

정답 요지:

- `name = p.name;`은 주소만 복사하는 얕은 복사
- 두 객체가 같은 메모리를 가리켜 소멸 시 이중 해제 위험이 생김

핵심 수정 코드:

```cpp
name = new char[strlen(p.name) + 1];
strcpy(name, p.name);
```

배점:

- 잘못된 줄 찾기 2점
- 이유 설명 2점
- 핵심 수정 코드 4점

허용:

- 중괄호 포함 여부는 무관
- 공백 차이는 무관

### 22번. [중] 문자열 나누기 코드 작성 (6점)

정답 예:

```cpp
(1) int pos = s.find(':');
(2) cout << s.substr(0, pos) << '\n';
(3) cout << s.substr(pos + 1) << '\n';
```

배점:

- (1) 1점
- (2) 1.5점
- (3) 1.5점

소계: 24점
