# Chapter 6 Summary

주제: 함수 중복(오버로딩), 디폴트 매개변수, 호출 모호성, static 멤버

이 문서는 실습에 들어가기 전에 6장의 핵심 개념을 한 번에 훑기 위한 요약입니다.

## 1. 오버로딩(Overloading)

오버로딩은 같은 이름의 함수를 여러 개 두는 기능입니다.
단, 컴파일러가 함수를 구분할 수 있도록 매개변수의 개수나 타입이 달라야 합니다.

예:
- `sum(int, int)`
- `sum(int, int, int)`
- `sum(double, double)`

핵심:
- 함수 이름이 같아도 괜찮다.
- 매개변수 리스트가 다르면 된다.
- 리턴 타입만 다른 것은 오버로딩이 아니다.

왜 중요한가:
- 비슷한 기능을 같은 이름으로 묶어 코드 의미를 자연스럽게 만들 수 있다.
- 다만 같은 이름에 너무 많은 의미를 넣으면 오히려 헷갈릴 수 있다.

## 2. 디폴트 매개변수(Default Parameter)

디폴트 매개변수는 인수를 생략했을 때 자동으로 들어가는 기본값입니다.

예:

```cpp
void star(int count = 5);
```

그러면:
- `star();` 는 `star(5);`처럼 동작한다.
- `star(10);` 은 전달한 값 10을 사용한다.

핵심:
- 보통 선언부에 적는다.
- 생략된 인수는 디폴트 값으로 채워진다.
- 디폴트 값은 뒤에서부터 연속으로만 줄 수 있다.

왜 중요한가:
- 비슷한 함수를 여러 개 만들지 않고 하나로 줄일 수 있다.
- 생성자에도 적용할 수 있어 클래스 설계를 단순하게 만들 수 있다.

## 3. 호출 모호성(Ambiguity)

오버로딩은 편리하지만, 컴파일러가 어느 함수를 골라야 할지 분명하지 않으면 에러가 납니다.

대표적인 경우:
- 오버로딩 + 디폴트 매개변수 충돌
- 여러 형 변환 후보가 동시에 가능한 경우
- 값 전달과 참조 전달을 같은 이름으로 오버로딩한 경우

예:
- `msg(int)` 와 `msg(int, string = "")`
- `square(float)` 와 `square(double)` 에서 `square(3)`
- `add(int, int)` 와 `add(int, int&)`

핵심:
- "둘 다 가능하다"면 컴파일러는 선택하지 못한다.
- 편한 설계보다 분명한 설계가 더 중요할 때가 많다.
- 의미가 다른 함수는 이름을 분리하는 편이 안전하다.

## 4. static 멤버 변수

일반 멤버 변수는 객체마다 따로 존재합니다.
반면 static 멤버 변수는 클래스 전체에서 하나만 존재하고 모든 객체가 공유합니다.

예:

```cpp
class Person {
public:
    int money;
    static int sharedMoney;
};
```

핵심:
- `money`는 객체마다 다르다.
- `sharedMoney`는 모든 객체가 함께 쓴다.
- static 멤버 변수는 클래스 밖에서 한 번 정의해야 한다.

예:

```cpp
int Person::sharedMoney = 10;
```

왜 중요한가:
- 객체 수와 관계없이 하나만 있어야 하는 값을 표현할 수 있다.
- 공용 설정값, 누적 카운트, 공유 데이터에 자주 사용된다.

## 5. static 멤버 함수

static 멤버 함수는 특정 객체가 아니라 클래스에 속한 함수입니다.

예:

```cpp
class Math {
public:
    static int abs(int a) { return (a >= 0) ? a : -a; }
};
```

호출:

```cpp
Math::abs(-5);
```

핵심:
- 객체를 만들지 않고 호출할 수 있다.
- 객체 상태가 필요 없는 함수를 묶기에 좋다.
- 전역 함수를 클래스 안으로 정리하는 효과가 있다.

## 6. static 함수의 제한

static 함수는 객체 없이 호출되므로 `this` 포인터가 없습니다.
따라서 non-static 멤버에 직접 접근할 수 없습니다.

즉:
- static 함수는 객체별 데이터를 바로 읽을 수 없다.
- 필요하면 객체를 인자로 받아야 한다.

이 제한을 이해해야:
- static을 어디에 써야 하는지
- 어디에는 쓰면 안 되는지
를 구분할 수 있습니다.

## 7. 6장에서 꼭 기억할 문장

- 오버로딩은 매개변수로 구분한다.
- 리턴 타입만 달라도 오버로딩이 되지는 않는다.
- 디폴트 매개변수는 뒤에서부터 연속으로만 줄 수 있다.
- 편리한 오버로딩이 항상 좋은 설계는 아니다.
- static 변수는 클래스 전체가 공유한다.
- static 함수에는 `this`가 없다.
