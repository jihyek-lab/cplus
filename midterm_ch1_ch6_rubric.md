## 1번. 프로그램 기본 구성 설명 (5점)

- `#include <iostream>`: 표준 입출력 사용 준비 1점
- `using namespace std;`: `std::` 생략 목적 1점
- `int main()`: 프로그램 시작 함수 2점
- `return 0;`: 정상 종료 의미 1점

## 2번. cout / 연산 / 출력 예측 (5점)

정답:

```text
8 15
sum=8
```

- 첫 줄 맞음 3점
- 둘째 줄 맞음 2점

## 3번. 입력과 버퍼 문제 설명 (8점)

- `cin >>` 뒤에 개행 문자 `\n`가 버퍼에 남는다고 설명 4점
- `cin.ignore(...)` 또는 이에 준하는 적절한 해결 코드 제시 4점

## 4번. C-문자열과 string 비교 (8점)

- `char[]`는 문자 배열, `string`은 클래스/객체라는 설명 3점
- `string`이 관리와 사용이 더 편리하다는 설명 1점
- `char[]`의 `==`는 내용이 아니라 주소/배열 비교 문제라는 취지 2점
- `strcmp()`로 내용 비교 설명 2점

## 5번. 문자열 처리 짧은 구현 (8점)

예시:

```cpp
string line;
getline(cin, line);
cout << line << '\n';
```

- `string` 선언 2점
- `getline()` 사용 3점
- 출력 3점

## 6번. 클래스와 캡슐화 설명 (8점)

- `private`은 외부 직접 접근 제한 3점
- `public`은 외부에 공개되는 인터페이스 2점
- 캡슐화의 목적: 데이터 보호, 잘못된 사용 방지, 내부 구현 은닉 3점

## 7번. 생성자와 소멸자 해석 (10점)

정답 순서:

```text
A ctor
A ctor
A dtor
A ctor
A dtor
A dtor
```

- 출력 순서 6점
- 생성은 선언 시점, 소멸은 스코프 종료 시점 설명 2점
- 지역 객체는 생성의 역순으로 소멸 설명 2점

## 8번. Circle 클래스 구현 (10점)

- `radius` private 1점
- 생성자 `Circle(int r = 1)` 2점
- 0 이하 방어 처리 2점
- `setRadius(int)` 2점
- `getRadius() const` 2점
- 전체 구조 자연스러움 1점

## 9번. struct와 class 차이 설명 (5점)

- 둘 다 멤버와 함수를 가질 수 있다는 공통점 2점
- 기본 접근 지정 차이: struct는 public, class는 private 3점

## 10번. 객체 배열과 기본 생성자 (8점)

- 배열을 만들면 각 원소 객체가 자동으로 생성된다는 설명 4점
- 인수 없이 생성될 수 있어야 해서 기본 생성자가 필요한 경우가 있다는 설명 4점

## 11번. new / delete 해석 (8점)

- `new int[5]`는 배열 할당이라는 설명 2점
- `delete p;`는 잘못이며 `delete[] p;`가 맞다는 설명 4점
- 잘못 해제 시 동작 오류 가능성 언급 2점

## 12번. this 포인터 설명 (8점)

- `this`는 현재 객체를 가리키는 포인터 4점
- `this->radius = radius;`에서 왼쪽은 멤버, 오른쪽은 매개변수 구분 설명 4점

## 13번. string 활용 코드 해석 (8점)

정답:

```text
130
```

- 출력 결과 4점
- `stoi()`는 문자열을 정수로 바꾸는 함수 설명 4점

## 14번. 동적 객체 짧은 구현 (10점)

예시 요소:

```cpp
Circle* p = new Circle;
p->setRadius(10);
cout << p->getRadius() << '\n';
delete p;
```

- 동적 생성 3점
- 반지름 설정 2점
- 출력 2점
- `delete`로 해제 3점

## 15번. 값/주소/참조 전달 비교 (10점)

- 값 전달: 복사본 사용, 원본 변경 안 됨 3점
- 주소 전달: 포인터 통해 원본 변경 가능 3점
- 참조 전달: 별명으로 원본 변경 가능 3점
- 호출 문법/사용 상황 설명 1점

## 16번. 참조 반환 설명 (8점)

- 참조 반환의 의미 설명 4점
- 반환값이 원본의 별명처럼 동작한다는 설명 2점
- `arrElem(a, 2) = 100;`가 가능한 이유 설명 2점

## 17번. 복사 생성자 호출 상황 (8점)

대표 정답:

- 객체를 값으로 전달할 때
- 객체를 값으로 반환할 때
- 기존 객체로 새 객체를 초기화할 때

- 1개당 4점, 최대 8점

## 18번. 얕은 복사와 깊은 복사 설명 (10점)

- 얕은 복사는 주소만 복사한다는 설명 3점
- 이중 해제, 공유로 인한 문제 설명 3점
- 깊은 복사는 새로운 메모리를 따로 확보해 내용을 복사한다는 설명 4점

## 19번. 참조 기반 함수 구현 (10점)

예시:

```cpp
void swapByRef(int& a, int& b) {
    int temp = a;
    a = b;
    b = temp;
}
```

- 함수 형식 3점
- 참조 사용 3점
- swap 로직 4점

## 24번. Counter 클래스 구현 (12점)

- `value` 일반 멤버 1점

- 생성자 증가 2점
- 소멸자 감소 2점
- `setValue(int)` 1점
- `getValue() const` 2점
- `static getNumOfObjects()` 1점

## 25번. static 함수의 제한 설명 (8점)

- static 함수는 객체 없이 호출 가능 2점
- 따라서 `this`가 없음 3점
- non-static 멤버는 객체별 데이터라 직접 접근 불가 설명 3점
